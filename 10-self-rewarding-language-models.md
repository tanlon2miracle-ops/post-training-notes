## 一、概览：三篇论文的关系

| 论文 | 机构 | 会议 | 核心贡献 | 缩写 |
|------|------|------|---------|------|
| Self-Rewarding Language Models | Meta | ICML 2024 | 提出自我奖励框架：模型自己打分 + 自己训练 + 迭代进化 | SRLM (原始) |

## 二、Meta: Self-Rewarding Language Models (ICML 2024)

### 2.1 核心问题

传统 RLHF 的两个瓶颈：
1. **人类标注天花板** — 奖励模型从人类偏好学习，能力被人类水平限制
2. **奖励模型冻结** — 训练好的 Reward Model 在 LLM 训练过程中不再更新，无法共同进化

### 2.2 核心方法：Iterative DPO + LLM-as-a-Judge

```
训练循环（每轮迭代）:

┌─────────────────────────────────────────────────────┐
│  Step A: 自己出题自己答                               │
│  给模型一批 prompt → 生成 N 个候选回答（多次采样）     │
├─────────────────────────────────────────────────────┤
│  Step B: 自己给自己打分（LLM-as-a-Judge）             │
│  同一个模型用 Judge prompt 对每个回答打 1-5 分         │
│  评分维度：相关性 / 覆盖度 / 有用性 / 清晰度 / 专业度  │
│  （5 个维度各 0-1 分，加总）                           │
├─────────────────────────────────────────────────────┤
│  Step C: 构造偏好对                                   │
│  高分回答 = chosen（好答案）                           │
│  低分回答 = rejected（差答案）                         │
│  组成 (prompt, chosen, rejected) 偏好对               │
├─────────────────────────────────────────────────────┤
│  Step D: DPO 训练                                    │
│  用标准 DPO loss 更新模型参数                         │
│  reference model = 上一轮的模型 M_t                   │
│  训练完得到 M_{t+1}                                   │
├─────────────────────────────────────────────────────┤
│  重复：M₀ → M₁ → M₂ → M₃                           │
└─────────────────────────────────────────────────────┘
```

### 2.3 DPO Loss 公式

$$\mathcal{L}_{DPO} = -\log\sigma\left(\beta \log\frac{\pi_\theta(y_w|x)}{\pi_{ref}(y_w|x)} - \beta \log\frac{\pi_\theta(y_l|x)}{\pi_{ref}(y_l|x)}\right)$$

其中：
- $y_w$ = chosen（好回答）, $y_l$ = rejected（差回答）
- $\pi_\theta$ = 当前模型, $\pi_{ref}$ = 上一轮模型
- $\beta$ = KL 散度控制系数

### 2.4 关键设计细节

| 设计 | 说明 |
|------|------|
| **Judge Prompt 模板** | 要求模型输出结构化评分（逐维度打分 + 总分），不是简单好坏 |
| **双重能力同时提升** | 每轮迭代不仅回答质量提升，打分能力也在提升——核心发现 |
| **无需外部奖励模型** | 不用单独训练 Reward Model，模型自己当 judge |
| **Reference model** | 用上一轮模型（M_t 作为 M_{t+1} 训练时的 ref） |
| **种子数据** | Open Assistant + 评判指令数据，对 Llama 2 70B 做 SFT |

### 2.5 关键发现

- **不仅指令遵循能力提升，自评能力也在迭代中提升** — 奖励模型不再是冻结的，而是共同进化的
- 3 轮迭代后（M₃）在 AlpacaEval 2.0 上**超过 Claude 2、Gemini Pro、GPT-4 0613**
- 打分能力（与人类标注一致性）随迭代**单调提升**

### 2.6 一句话总结

> 模型自己生成回答 → 自己打分 → 用打分构造偏好对 → DPO 训练 → 循环。不需要人类标注，两个能力同时变强。

### 2.7 开源代码

| 来源 | 链接 | 说明 |
|------|------|------|
| 社区复现 (Oxen.AI) | https://github.com/Oxen-AI/Self-Rewarding-Language-Models | 完整 5 步脚本，基于 Mistral-7B |
| 社区复现 (raulc0399) | https://github.com/raulc0399/self-rewarding-language-models | 另一个复现 |
| ⚠️ Meta 官方 | **未开源代码** | — |

## 三、CREAM: Consistency Regularized Self-Rewarding (ICLR 2025)

### 3.1 核心问题

Meta 的 SRLM 在**小模型（7B）上迭代几轮后效果饱和甚至退化**。

原因：**自评分的偏差会累积**。模型打分不够准 → 偏好数据不可靠 → 训练出的模型打分更不准 → 恶性循环。

### 3.2 核心方法：跨迭代一致性正则化

```
CREAM 的改进思路：

迭代 t 的模型 M_t 对回答 y 打分 → r_t(y)
迭代 t+1 的模型 M_{t+1} 对回答 y 打分 → r_{t+1}(y)

比较两轮评分：
  ├─→ 一致（两轮都觉得好/都觉得差）→ 可靠！保留正常权重
  └─→ 不一致（前后打分矛盾）→ 不可靠！降低这条数据的权重

数学实现：在 DPO loss 中加入基于奖励一致性的正则项
```

### 3.3 效果

- 同时提升了**奖励一致性**和**对齐性能**
- 解决了小模型自我奖励退化的问题
- 在 7B 参数模型上效果尤其显著

### 3.4 简单对比

| 维度 | SRLM (Meta) | CREAM |
|------|------------|-------|
| 适用模型 | 大模型 (70B) 效果好 | 小模型 (7B) 也能持续提升 |
| 打分质量保证 | 无（靠迭代自然提升） | 跨迭代一致性正则化 |
| 偏好数据可靠性 | 可能退化 | 不确定的分数少用 |
| 收敛性 | 大模型可持续，小模型饱和 | 防止自嗨退化 |

### 3.5 开源代码

✅ **官方开源：** https://github.com/Raibows/CREAM

## 四、自我奖励的其他防退化机制

除了 CREAM，学界还提出了几种防止"自嗨"的方法：

| 方法 | 机制 |
|------|------|
| **CREAM** | 跨迭代一致性正则化，惩罚前后打分不一致的行为 |
| **SCIR** | 置信度过滤，只在内部多个评分器达成共识时才更新 |
| **信息熵约束** | 防止模型对自己过度自信 |
| **KL 散度正则化** | 标准 RLHF 中的 KL penalty，防止模型偏离太远 |

**理论保证：** 即使初始模型很弱（打分不准），迭代过程会**指数级缩小初始误差**的影响，最终收敛到 O(1/√n) 的标准速率。

## 五、Apple: SRLM - Self-Reflective Program Search (2026 preprint)

### 5.1 核心问题

大模型现在能处理很长的文本（几十万甚至上百万 token），但实际上**越长越容易犯迷糊**——中间的关键信息丢了、被无关内容干扰了、推理断链了。

### 5.2 前置工作：RLM（递归语言模型）

RLM 的做法：不让模型一口气读完长文本，而是让它**写程序来查询文本**——就像写代码一样，一段一段地切、查、递归调用自己，把大问题拆成小问题。

**RLM 的缺陷：** 怎么选"查哪一段、怎么查"全靠固定规则，没有判断好坏的能力。就像一个人在图书馆按固定规则翻资料，但从不停下来想"我找对了吗？"

### 5.3 SRLM 的核心创新：自反思搜索

```
核心思路：让模型自己判断"我这次查对了没有"

                    ┌─────────────────┐
                    │   给定长文本 +   │
                    │    一个问题      │
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │ 生成多条候选的    │
                    │ "上下文交互程序"  │
                    │ （不同的切分方式、│
                    │  不同的子问题）   │
                    └────────┬────────┘
                             │
              ┌──────────────┼──────────────┐
              │              │              │
    ┌─────────▼─────┐ ┌─────▼──────┐ ┌─────▼──────────┐
    │ 自一致性      │ │ 推理链长度  │ │ 口头置信度      │
    │ Self-         │ │ Reasoning  │ │ Verbalized     │
    │ Consistency   │ │ Length     │ │ Confidence     │
    │               │ │            │ │                │
    │ 同一问题跑    │ │ 太短=没想  │ │ 直接问模型      │
    │ 多次，答案    │ │ 清楚       │ │ "你有多确定？"  │
    │ 一致的更可信  │ │ 太长=跑偏  │ │ 让它给自己打分  │
    └───────┬───────┘ └─────┬──────┘ └────────┬───────┘
              │              │              │
              └──────────────┼──────────────┘
                             │
                    ┌────────▼────────┐
                    │ 综合三个信号打分  │
                    │ 选出最佳推理路径  │
                    │ 作为最终答案      │
                    └─────────────────┘
```

### 5.4 三个自反思信号

| 信号 | 原理 | 为什么有效 |
|------|------|----------|
| **Self-Consistency** | 同一问题多次采样，答案一致性越高越可信 | 稳定的答案更可能是对的 |
| **Reasoning Length** | 推理链长度作为不确定性指标 | 太短=没想清楚，太长=可能跑偏 |
| **Verbalized Confidence** | 直接问模型"你有多确定" | 模型的内部状态反映在输出中 |

**关键：** 这三个信号**不需要外部监督**，完全靠模型自身判断。

### 5.5 关键发现（比较颠覆）

| 发现 | 说明 |
|------|------|
| **递归本身不是关键** | 不需要 RLM 那样层层递归，光靠"生成多个方案 + 自我反思选最好的"就能持平甚至超过 RLM |
| **短文本上 RLM 反而掉分** | 如果文本长度在模型窗口内，RLM 的递归拆分反而引入不必要的噪音 |
| **SRLM 短和长都稳定提升** | 无论上下文长短，自反思搜索都带来收益 |
| **语义密集型任务 RLM 搞不定** | RLM 靠规则切分容易漏关键信息；SRLM 的自反思能提供更高层语义引导 |
| **同等时间预算下提升 22%** | SRLM 比 RLM 最高提升 22% |

### 5.6 类比

> **RLM** 就像一个人在图书馆找资料，按固定规则翻——先查目录、再翻第 X 页、再看第 Y 段。但从不停下来想"我找对了吗？"
>
> **SRLM** 是同样翻资料，但每翻完一轮会停下来问自己："我确定这是对的吗？多试几种翻法，看哪种结果最一致、最有信心，就用哪个。"

### 5.7 代码

⚠️ **Apple 官方未开源代码**（论文为 preprint，Apple 的论文通常代码开源较慢）

方法本身不复杂——核心就是 best-of-N sampling + 三个自评估信号，理论上用现有框架（vLLM + 简单 wrapper）就能复现。

## 六、总结与对比

### 三篇论文定位对比

| 维度 | Meta SRLM | CREAM | Apple SRLM |
|------|----------|-------|-----------|
| **解决问题** | 去掉人类标注的对齐训练 | 小模型自我奖励退化 | 长上下文推理不准 |
| **阶段** | 训练时 | 训练时 | 推理时 |
| **核心机制** | Iterative DPO + LLM-as-a-Judge | 跨迭代一致性正则化 | best-of-N + 自反思信号 |
| **自评方式** | 5 维结构化评分 | 同上 + 一致性过滤 | 自一致性 + 推理长度 + 口头置信度 |
| **基座模型** | Llama 2 70B | 7B 级模型 | 通用（论文测了多种） |
| **开源代码** | ❌ Meta 未开源（社区复现有） | ✅ 官方开源 | ❌ Apple 未开源 |

### "自我奖励"的核心洞察

1. **模型能当自己的裁判** — 不需要单独的 Reward Model，LLM 自己打分就够用
2. **打分能力会共同进化** — 迭代训练中，回答质量和打分质量同时提升
3. **但小模型会"自嗨"** — 偏差累积导致退化，需要正则化（CREAM）或置信度过滤
4. **自反思不只用于训练** — 推理时也能用自我评估信号来选择最佳推理路径（Apple SRLM）
